# 📄 GSD 데이터 구조 최종 정의 (참조 관계 보강판)

---

# 1. ConditionTable

Condition은 게임 내 상태를 판단하기 위한 단일 조건 데이터이다.

같은 `ConditionGroupID`를 가진 Condition들은 **모두 만족해야 하는 AND 조건 묶음**으로 처리한다.

| 컬럼 | 설명 | 참조 |
| --- | --- | --- |
| ConditionID | 조건 고유 ID (PK) | - |
| ConditionGroupID | 동일 조건 묶음 ID. 같은 값을 가진 행들은 AND로 함께 평가된다 | - |
| ConditionType | 어떤 종류의 상태를 검사할지 정의 | `EConditionType` |
| ConditionTargetID | 검사 대상 ID. ConditionType에 따라 의미가 달라지며, 대상이 없으면 0 사용 | ConditionType에 따라 CharacterID / EvidenceID / InvestigationID / 0 |
| CompareType | 비교 방식 | `ECompareType` |
| ConditionValue | 비교 기준값 | - |

## ConditionTargetID 해석 규칙

| ConditionType | ConditionTargetID 의미 |
| --- | --- |
| Trust | `CharacterTable.CharacterID` |
| EvidenceOwned | `EvidenceTable.EvidenceID` |
| RevealedCharacter | `CharacterTable.CharacterID` |
| SceneProgressIndex | 0 |
| InvestigationScore | `InvestigationTable.InvestigationID` |
| ResonanceLevel | 0 |

---

# 2. SceneTable

Scene는 장면 단위 메타 데이터를 정의한다.

직접적인 흐름 분기 책임은 가지지 않으며, Scene 종료 후 이동은 BranchTable에서 처리한다.

| 컬럼 | 설명 | 참조 |
| --- | --- | --- |
| SceneID | 씬 고유 ID (PK) | - |
| Chapter | 챕터 번호 | - |
| Title | 씬 제목 | - |
| Background | 배경 이미지 경로 | 리소스 경로 |
| Music | BGM 경로 | 리소스 경로 |
| Effect | 씬 진입 시 재생할 연출 효과 | Effect 리소스 또는 enum |
| GoalKicker | 목표 UI 머리말 | - |
| GoalText | 현재 목표 문장 | - |
| EvidencePromptTitle | 증거 제시 UI 제목 | - |
| EvidencePromptHint | 증거 제시 UI 안내 문구 | - |

---

# 3. DialogTable

Dialog는 Scene 내부에서 순차적으로 출력되는 대사 및 흐름 노드이다.

기본적으로 `Order` 순으로 진행되며, 필요 시 `NextDialogID`로 특정 Dialog를 명시적으로 연결할 수 있다.

| 컬럼 | 설명 | 참조 |
| --- | --- | --- |
| DialogID | 대사 고유 ID (PK) | - |
| SceneID | 소속 Scene ID | `SceneTable.SceneID` |
| Order | 동일 Scene 내 기본 출력 순서 | - |
| CharacterID | 화자 캐릭터 ID. 나레이션 등 화자가 없을 경우 0 또는 빈값 사용 규칙 필요 | `CharacterTable.CharacterID` |
| EmotionType | 화자의 감정 상태. 해당 감정 이미지가 없으면 `Default` 사용 | `CharacterEmotionTable.(CharacterID, EmotionType)` |
| StandingSlot | 화면 내 캐릭터 표시 위치 | `EStandingSlot` |
| FocusType | 현재 대사의 포커스 방식 | `EFocusType` |
| EnterMotion | 등장 모션 | Motion enum 또는 리소스 키 |
| ExitMotion | 퇴장 모션 | Motion enum 또는 리소스 키 |
| IdleMotion | 유지 모션 | Motion enum 또는 리소스 키 |
| FxType | 즉시 재생 FX | FX enum 또는 리소스 키 |
| Text | 대사 본문 | - |
| Style | 대사 스타일 (일반/나레이션/생각 등) | `EDialogStyle` |
| ConditionGroupID | 이 대사를 표시하기 위한 조건 묶음 ID. 값이 없으면 항상 표시 | `ConditionTable.ConditionGroupID` |
| ChoiceGroupID | 이 대사 시점에 호출할 선택지 그룹 ID | `ChoiceGroupTable.ChoiceGroupID` |
| NextDialogID | 다음으로 강제 이동할 Dialog ID. 없으면 Order 기반 진행 | `DialogTable.DialogID` |

---

# 4. ChoiceGroupTable

ChoiceGroup은 한 번에 표시되는 선택지 묶음을 정의한다.

선택지 그룹 자체의 종류와 노출 조건, 선택 가능 개수를 관리한다.

| 컬럼 | 설명 | 참조 |
| --- | --- | --- |
| ChoiceGroupID | 선택지 그룹 고유 ID (PK) | - |
| Type | 선택지 그룹 종류 (일반 / 조사 / 증거) | `EChoiceGroupType` |
| ConditionGroupID | 이 선택지 그룹을 표시하기 위한 조건 묶음 ID. 값이 없으면 항상 표시 | `ConditionTable.ConditionGroupID` |
| MaxSelectable | 동시에 선택 가능한 개수 | - |

---

# 5. ChoiceTable

Choice는 실제 플레이어가 고르는 개별 선택지이다.

어떤 그룹에 속하는지, 어떤 순서로 출력되는지, 선택 시 어디로 이동하는지를 정의한다.

| 컬럼 | 설명 | 참조 |
| --- | --- | --- |
| ChoiceID | 선택지 고유 ID (PK) | - |
| ChoiceGroupID | 선택지 그룹 ID | `ChoiceGroupTable.ChoiceGroupID` |
| Order | 동일 그룹 내 출력 순서 | - |
| Text | 선택지 표시 텍스트 | - |
| ConditionGroupID | 선택지 노출 조건 (없으면 항상 표시) | `ConditionTable.ConditionGroupID` |
| NextType | 선택 후 이동 타입 | `ENextType (Scene / Dialog / None)` |
| NextID | 이동 대상 ID | `SceneTable.SceneID` 또는 `DialogTable.DialogID` |
| EvidenceID | 획득할 증거 ID (없으면 0) | `EvidenceTable.EvidenceID` |
| TrustCharacterID | 신뢰도 대상 캐릭터 ID (없으면 0) | `CharacterTable.CharacterID` |
| TrustValue | 신뢰도 변화값 (증가/감소) | - |
| ResonanceValue | 공명도 변화값 | - |

## NextID 해석 규칙

| NextType | NextID 의미 |
| --- | --- |
| Scene | `SceneTable.SceneID` |
| Dialog | `DialogTable.DialogID` |
| None | 0 또는 빈값 |

---

# 6. BranchTable

Branch는 Scene 종료 시 자동으로 평가되는 분기 데이터이다.

같은 Scene에 속한 Branch를 `Order` 순으로 검사하고, 가장 먼저 조건을 만족한 Scene으로 이동한다.

| 컬럼 | 설명 | 참조 |
| --- | --- | --- |
| BranchID | 분기 고유 ID (PK) | - |
| SceneID | 이 분기가 속한 Scene ID | `SceneTable.SceneID` |
| Order | 분기 평가 순서 | - |
| ConditionGroupID | 분기 조건 묶음 ID | `ConditionTable.ConditionGroupID` |
| NextSceneID | 조건 만족 시 이동할 다음 Scene ID | `SceneTable.SceneID` |

---

# 7. EvidenceTable

Evidence는 게임 내에서 획득 가능한 증거의 정적 정의 데이터이다.

보유 여부는 이 테이블이 아니라 런타임 상태에서 관리한다.

| 컬럼 | 설명 | 참조 |
| --- | --- | --- |
| EvidenceID | 증거 고유 ID (PK) | - |
| Name | 증거 이름 | - |
| Description | 증거 설명 | - |
| Image | 증거 이미지 경로 | 리소스 경로 |
| CategoryID | UI 그룹화를 위한 카테고리 ID | `EvidenceCategoryTable.CategoryID` |

---

# 8. EvidenceCategoryTable

EvidenceCategory는 증거를 UI에서 묶어 보여주기 위한 그룹 정의 데이터이다.

게임 진행 로직에는 직접 관여하지 않는다.

| 컬럼 | 설명 | 참조 |
| --- | --- | --- |
| CategoryID | 카테고리 고유 ID (PK) | - |
| CategoryTitle | 카테고리 이름 | - |
| CategoryHint | 카테고리 설명 | - |

---

# 9. CharacterTable

Character는 캐릭터의 기본 정보와 수첩 표시용 정보를 관리한다.

이미지와 감정 표현은 CharacterEmotionTable에서 별도로 관리한다.

| 컬럼 | 설명 | 참조 |
| --- | --- | --- |
| CharacterID | 캐릭터 고유 ID (PK) | - |
| DisplayName | 캐릭터 표시 이름 | - |
| NotebookSummary1 | 수첩 요약 문장 1 | - |
| NotebookSummary2 | 수첩 요약 문장 2 | - |

---

# 10. CharacterEmotionTable

CharacterEmotion은 캐릭터별 감정 상태에 따른 이미지 매핑 데이터이다.

기본 이미지는 `EmotionType = Default` 행을 통해 제공한다.

| 컬럼 | 설명 | 참조 |
| --- | --- | --- |
| CharacterID | 캐릭터 ID | `CharacterTable.CharacterID` |
| EmotionType | 감정 타입. 반드시 `Default` 행이 존재해야 함 | `EEmotionType` |
| ImagePath | 감정별 초상화 경로 | 리소스 경로 |

## PK 권장

- `(CharacterID, EmotionType)` 복합 유니크

---

# 11. InvestigationTable

Investigation은 조사 시스템의 기본 정보를 정의한다.

조사 제목, 안내 문구, 조사 가능 횟수 및 사용할 선택지 그룹을 가진다.

| 컬럼 | 설명 | 참조 |
| --- | --- | --- |
| InvestigationID | 조사 고유 ID (PK) | - |
| Title | 조사 제목 | - |
| Hint | 조사 안내 문구 | - |
| Budget | 조사 가능 횟수 | - |
| ChoiceGroupID | 해당 조사에서 사용할 선택지 그룹 ID | `ChoiceGroupTable.ChoiceGroupID` |

---

# 12. Enum / 참조 타입 정리

## EConditionType

- Trust
- EvidenceOwned
- RevealedCharacter
- SceneProgressIndex
- InvestigationScore
- ResonanceLevel

## ECompareType

- Equal
- NotEqual
- Greater
- GreaterEqual
- Less
- LessEqual

## EChoiceGroupType

- Normal
- Investigation
- Evidence

## ENextType

- Scene
- Dialog
- None

## EEmotionType

- Default
- 기타 감정값

# 13. 참조 관계 한눈에 보기

- `DialogTable.SceneID` → `SceneTable.SceneID`
- `DialogTable.CharacterID` → `CharacterTable.CharacterID`
- `DialogTable.ChoiceGroupID` → `ChoiceGroupTable.ChoiceGroupID`
- `DialogTable.NextDialogID` → `DialogTable.DialogID`
- `DialogTable.ConditionGroupID` → `ConditionTable.ConditionGroupID`
- `ChoiceTable.ChoiceGroupID` → `ChoiceGroupTable.ChoiceGroupID`
- `ChoiceTable.NextID` → `SceneTable.SceneID` or `DialogTable.DialogID` (by `NextType`)
- `ChoiceTable.ConditionGroupID` → `ConditionTable.ConditionGroupID`
- `ChoiceGroupTable.ConditionGroupID` → `ConditionTable.ConditionGroupID`
- `BranchTable.SceneID` → `SceneTable.SceneID`
- `BranchTable.NextSceneID` → `SceneTable.SceneID`
- `BranchTable.ConditionGroupID` → `ConditionTable.ConditionGroupID`
- `EvidenceTable.CategoryID` → `EvidenceCategoryTable.CategoryID`
- `CharacterEmotionTable.CharacterID` → `CharacterTable.CharacterID`
- `InvestigationTable.ChoiceGroupID` → `ChoiceGroupTable.ChoiceGroupID`
