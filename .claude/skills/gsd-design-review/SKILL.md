# GSD Design Review

Use this skill for visual review only.

Focus on:

- hierarchy
- readability
- spacing
- alignment
- density
- consistency
- placeholder quality
- visual fit with the reference portfolio family

Do not debate project claims here. That belongs to `gsd-content-review`.

## Goal

Catch the visual issues that make a slide feel untrustworthy, unfinished, hard to read, or outside the reference portfolio family.

## Review order

1. can it be read fast
2. does hierarchy hold
3. does it feel reference-fit
4. is layout stable
5. do repeated patterns stay consistent
6. only then cosmetic polish

## Hard rules

- do not suggest flashy redesigns by default
- do not ask for more decoration just to fill space
- do not force image-heavy layouts
- placeholders are allowed if they look intentional
- protect `Noto Sans KR` usage
- judge the slide as a hiring document first
- treat `Ref` as the target family for clarity and professionalism

## Required checks

### 1. Readability

Flag:

- text too small
- low contrast
- overlong lines
- dense blocks with no breathing room
- footer or caption text that becomes useless at export size

### 2. Reference-fit

Ask:

- does the slide feel like a portfolio page, not a draft board
- do images behave like evidence, not decoration
- is each slide message simpler and more controlled like the reference decks

Flag slides that feel too empty, too decorative, or too loose compared with the references.

### 3. Hierarchy

Flag when:

- title, body, and summary feel too similar
- accent color is overused
- everything has equal weight
- cards or images compete with the main message

### 4. Layout stability

Flag:

- misalignment
- uneven card sizes without reason
- awkward image crops
- objects touching margins
- elements that look like accidental overlap

### 5. Placeholder quality

Allowed, but only if:

- clearly labeled
- proportionally clean
- visually consistent with real assets

Flag placeholders that look like missing work rather than planned structure.

### 6. Consistency

Check repeated systems:

- title band
- page number position
- footer pattern
- card radius/shadow
- placeholder style
- image framing

## Output format

Use this exact order:

1. `blocking visual issues`
2. `high-value fixes`
3. `optional polish`
4. `working well`

Rules:

- one issue per line
- mention slide number
- say the concrete visual problem, not vague taste
- keep `optional polish` short

## Good finding examples

- `blocking visual issues: Slide 7 right-side cards are readable on screen but too dense for PDF export at normal zoom.`
- `high-value fixes: Slide 2 images feel decorative, while the reference decks use visuals more explicitly as proof or explanation.`
- `high-value fixes: Slide 1 hero image frame is so dominant that it pulls attention before the title does.`
- `optional polish: Slide 10 could align the three evidence areas to a stricter grid.`

## Done when

- unreadable slides are identified
- reference-family mismatches are identified
- hierarchy problems are clear
- placeholder quality is judged explicitly
- output is short enough to use as an action list
