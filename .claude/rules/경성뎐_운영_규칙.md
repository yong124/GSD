# 경성뎐 운영 규칙

이 저장소에서 작업할 때는 아래 규칙을 우선 적용한다.

## 1. 원본과 편집 기준

- 빠른 실무 수정의 현재 기준 원본은 `game/data/game_data.js`다.
- `content/data/script.xlsx`는 원본 테이블이지만, 현재 운영에선 대량 수정 / 검수 / 공유용 포맷으로 본다.
- `content/generated/script.generated.xlsx`는 보조 산출물이지 장기 원본이 아니다.
- `export_to_json.py`를 실행하면 `game_data.js`가 덮어써질 수 있으므로, 지금 어느 파일을 기준으로 작업 중인지 항상 명확히 한다.

## 2. 한국어 데이터 수정 규칙

- 한국어 데이터는 가능하면 `apply_patch`로 수정한다.
- PowerShell / shell / node 인라인 문자열로 한국어 대량 삽입을 하지 않는다.
- 터미널에서 한글이 깨져 보여도 바로 파일 손상으로 판단하지 않는다.
- 먼저 아래 방식으로 UTF-8 기준 재확인한다.

```powershell
chcp 65001 > $null
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
Get-Content '경로' -Encoding utf8
```

## 3. 브라우저 반영 규칙

- `game/index.html`에서 로드되는 `CSS/JS`를 수정했으면 `?v=` 리소스 버전도 함께 올린다.
- 특히 아래 구간은 캐시 영향이 크므로 버전 갱신을 기본으로 한다.
  - 타이틀
  - 저장/불러오기
  - 메모장
  - HUD
  - 씬 전환 / UI 패널

## 4. 수정 후 기본 검증

- 데이터 수정 후:

```powershell
py G:\GSD\content\tools\validate_game_data.py
```

- 런타임 JS 수정 후:

```powershell
node --check G:\GSD\game\js\engine\<파일명>.js
```

- 필요 시 generated 갱신:

```powershell
py G:\GSD\content\tools\json_to_generated_xlsx.py
```

## 5. UI 입력 흐름 규칙

- 타이틀, 저장 패널, 메모장 패널, 선택지 UI를 수정했다면 아래도 같이 본다.
  - `새 게임`
  - `이어하기`
  - `Esc`
  - `M / S / L`
  - 패널이 열렸을 때 대사 진행이 막히는지

## 6. 현재 프로젝트 방향

- 목표는 "비주얼노벨 같은 텍스트 플레이"가 아니라 "조사형 시스템이 보이는 ADV"다.
- 따라서 작업 우선순위는 아래 순서를 따른다.
  1. 플레이 의미가 더 또렷해지는가
  2. 선택과 상태 변화가 더 읽히는가
  3. UI/연출이 그 변화를 더 잘 전달하는가
  4. 그 다음에 미감과 문장 폴리시
## Data-First Rule

- Prefer table-backed data over hardcoded runtime maps for repeatable gameplay content.
- Before adding new constants or switch chains, decide whether the content belongs in:
  - an existing table
  - a new table
  - or truly runtime-only derived logic
- Prefer existing table expansion before inventing a new table.
- Good candidates for table ownership:
  - notebook character summaries
  - evidence category metadata
  - scene goals
  - questions
  - state descriptors
  - visibility/state rules
- If a structure changes, update `EditorNode`, pipeline scripts, validation, docs, `game_data`, and runtime references together whenever possible.
- Leave logic in code only when it is derived from current play state rather than authored content.
