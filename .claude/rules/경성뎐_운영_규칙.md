# 경성뎐 운영 규칙

이 저장소에서 작업할 때는 아래 규칙을 기본 기준으로 삼는다.

## 1. 기본 소스 오브 트루스

- 빠른 실무 수정의 현재 기준 파일은 `game/data/game_data.js`다.
- `content/data/script.xlsx`는 원본 테이블이지만, 현재 운영에서는 검토와 공유용 산출물에 가깝다.
- `content/generated/script.generated.xlsx`는 보조 검수 산출물이지 단일 원본이 아니다.
- `export_to_json.py`를 다시 돌리면 `game_data.js`가 덮어써질 수 있으므로, 지금 어느 파일을 기준으로 작업 중인지 먼저 분명히 한다.

## 2. 한국어 스킬 호출 별칭

스킬 폴더명과 내부 `name:`은 영문 ID를 유지한다. 대신 대화에서는 아래 한국어 이름으로 불러도 같은 작업 축으로 간주한다.

- `데이터 폴리시` -> `gsd-data-polish`
- `런타임 UI` -> `gsd-runtime-ui`
- `조사 설계` 또는 `조사 루프` -> `gsd-priority-investigation`
- `에디터노드 파이프라인` -> `gsd-editornode-pipeline`
- `브라우저 QA` -> `gsd-browser-qa`
- `릴리즈 체크` 또는 `마감 체크` -> `gsd-release-checks`

권장 방식:

- 작업 요청에 한국어 별칭을 직접 적는다.
- 한 번에 스킬은 보통 1개, 많아도 2개까지만 묶는다.
- 작업의 중심 축이 무엇인지 먼저 정한 뒤 그 스킬을 부른다.

예시:

- `브라우저 QA 기준으로 전체 씬 진행 막힘 검사해줘`
- `데이터 폴리시 기준으로 ch5 후반 대사 톤 정리해줘`
- `런타임 UI 기준으로 메모 패널 UX 손봐줘`
- `릴리즈 체크로 마감 정리하고 커밋 준비해줘`

## 3. 한글 데이터 수정 규칙

- 한글 텍스트 수정은 가능하면 `apply_patch`로 처리한다.
- PowerShell, shell, node 인라인 문자열로 한국어를 대량 삽입하지 않는다.
- 글자가 깨져 보이면 파일 손상으로 단정하지 말고 먼저 UTF-8로 다시 확인한다.

```powershell
chcp 65001 > $null
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
Get-Content '경로' -Encoding utf8
```

## 4. 브라우저 반영 규칙

- `game/index.html`에서 로드하는 `CSS` 또는 `JS`를 수정했으면 `?v=` 버전을 함께 올린다.
- 특히 아래 영역은 캐시 영향이 커서 버전 갱신을 기본으로 본다.
  - 타이틀
  - 이어하기
  - 메모
  - HUD
  - 패널 전환과 입력 흐름

## 5. 수정 후 기본 검증

- 데이터 수정 후: `py G:\GSD\content\tools\validate_game_data.py`
- 런타임 JS 수정 후: `node --check G:\GSD\game\js\engine\<파일명>.js`
- generated 갱신이 필요하면: `py G:\GSD\content\tools\json_to_generated_xlsx.py`

## 6. UI 입력 흐름 규칙

- 타이틀, 저장 패널, 메모 패널, 선택지 UI를 손댔다면 아래 항목을 함께 본다.
  - `새 게임`
  - `이어하기`
  - `Esc`
  - `M / S / L`
  - 패널이 열린 상태에서 대사 진행이 막히는지

## 7. 현재 프로젝트 우선순위

- 목표는 "텍스트 노벨 같은 정적 화면"이 아니라 조사와 상태 변화가 보이는 ADV다.
- 작업 우선순위는 아래 순서를 따른다.
  1. 플레이가 실제로 진행되는가
  2. 선택과 상태 변화가 읽히는가
  3. UI와 연출이 그 변화를 전달하는가
  4. 그다음 문장 밀도와 감정 정리를 본다

## 8. Data-first Rule

반복 가능한 게임플레이 메타데이터는 런타임 상수보다 테이블 이관을 먼저 검토한다.

권장 순서:

1. 기존 테이블 확장
2. 신규 테이블 추가
3. 파생 로직만 런타임 처리

구조를 바꾸면 가능하면 한 라운드 안에 아래를 같이 맞춘다.

- `EditorNode`
- `pipeline`
- `game_data`
- `runtime`
- `validate`
- `docs`
