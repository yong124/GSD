# 경성뎐 검증 체크리스트

작업 후에는 변경 축에 맞는 검증만 빠르게 고른다.

## 1. 데이터 수정

```powershell
py G:\GSD\content\tools\validate_game_data.py
```

필요하면:

```powershell
py G:\GSD\content\tools\json_to_generated_xlsx.py
```

## 2. 런타임 JS 수정

예시:

```powershell
node --check G:\GSD\game\js\engine\choice.js
node --check G:\GSD\game\js\engine\scene.js
node --check G:\GSD\game\js\main.js
```

수정한 파일만 골라서 확인한다.

## 3. EditorNode 수정

```powershell
node --check G:\GSD\EditorNode\editor.js
```

파이썬 툴도 바뀌었다면:

```powershell
py -m py_compile G:\GSD\content\tools\export_to_json.py G:\GSD\content\tools\json_to_generated_xlsx.py G:\GSD\content\tools\validate_game_data.py
```

## 4. 브라우저 캐시 체크

아래가 바뀌었으면 `game/index.html`의 `?v=`도 올렸는지 본다.

- `game/css/*`
- `game/js/*`

특히:

- 타이틀
- 저장/불러오기
- 메모장
- HUD
- 씬 전환 / UI 패널

## 5. 입력 흐름 체크

아래를 건드렸으면 같이 본다.

- 타이틀 버튼
- 저장 패널
- 메모장 패널
- 선택지 UI
- 대사 진행

확인 항목:

- `새 게임`
- `이어하기`
- `Esc`
- `M / S / L`
- 패널이 열린 상태에서 대사 진행 차단

## 6. 마감 전 정리

```powershell
git status --short
```

남겨두면 안 되는 것:

- QA 임시 폴더
- npm 캐시 폴더
- test artifacts
