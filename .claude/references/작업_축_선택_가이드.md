# 경성뎐 작업 축 선택 가이드

작업을 시작하기 전에 먼저 아래 질문으로 축을 고른다.

## 1. 주된 변경 대상이 무엇인가

### A. `game_data.js` 대사 / 분기 / 엔딩 / 단서 반응

먼저 사용할 스킬:

- `gsd-data-polish`

추가로 필요할 수 있는 스킬:

- `gsd-priority-investigation`
- `gsd-release-checks`

### B. HUD / 대화창 / 선택지 / 메모장 / 저장 패널 / 단축키

먼저 사용할 스킬:

- `gsd-runtime-ui`

추가로 필요할 수 있는 스킬:

- `gsd-release-checks`

### C. `priority_budget` 조사 씬 구조 / 루프 / branch pacing

먼저 사용할 스킬:

- `gsd-priority-investigation`

추가로 필요할 수 있는 스킬:

- `gsd-runtime-ui`
- `gsd-data-polish`
- `gsd-release-checks`

### D. EditorNode / export / generated / validate / 스키마

먼저 사용할 스킬:

- `gsd-editornode-pipeline`

추가로 필요할 수 있는 스킬:

- `gsd-release-checks`

### E. 마감 / handoff / 커밋 / validate / temp 정리

먼저 사용할 스킬:

- `gsd-release-checks`

## 2. 겹치는 경우 우선순위

### priority 씬을 새로 만드는 경우

1. `gsd-priority-investigation`
2. `gsd-data-polish`
3. `gsd-runtime-ui`
4. `gsd-release-checks`

### 메모장 / HUD / 저장 패널과 데이터가 같이 바뀌는 경우

1. `gsd-runtime-ui`
2. `gsd-data-polish` 또는 `gsd-priority-investigation`
3. `gsd-release-checks`

### Character / Emotion 구조가 바뀌는 경우

1. `gsd-editornode-pipeline`
2. `gsd-runtime-ui`
3. `gsd-release-checks`

## 3. 빠른 판단 규칙

- 화면은 안 바뀌고 문장/분기만 바뀐다: `gsd-data-polish`
- 조사 기회, priority loop가 핵심이다: `gsd-priority-investigation`
- 패널/HUD/입력 흐름이 핵심이다: `gsd-runtime-ui`
- EditorNode, generated, validate가 핵심이다: `gsd-editornode-pipeline`
- 마감 단계다: `gsd-release-checks`
