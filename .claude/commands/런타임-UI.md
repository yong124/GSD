# 런타임 UI

이 명령은 아래 순서로 움직인다.

1. 현재 사용자 루프를 정의
   - 타이틀 / 선택지 / 메모장 / 저장 / HUD / 조사
2. 최소한의 런타임 파일만 수정
3. `index.html`의 `?v=` 갱신 여부 확인
4. 수정한 JS 파일 `node --check`
5. 입력 충돌 항목 확인
   - `새 게임`
   - `이어하기`
   - `Esc`
   - `M / S / L`
   - 패널 열린 상태 대사 진행
6. 바뀐 UI 포인트와 남은 QA 포인트를 짧게 정리

우선 사용할 스킬:

- `gsd-runtime-ui`

필요 시 같이 사용할 스킬:

- `gsd-release-checks`
