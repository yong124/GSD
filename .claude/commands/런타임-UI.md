# 런타임 UI

이 명령은 HUD, 메모 패널, 저장 패널, 선택지 UI, 반응형 레이아웃, 입력 충돌 같은 브라우저 런타임 화면 작업에 쓴다.

## 기본 순서

1. 어떤 플레이 루프를 손보는지 먼저 정한다.
   - 타이틀
   - 대사 진행
   - 선택지
   - 조사 루프
   - 메모 또는 증거
   - 저장 또는 불러오기
2. 최소 범위의 런타임 파일만 수정한다.
3. 브라우저 로드 자산을 바꿨으면 `index.html`의 `?v=`를 올린다.
4. 수정한 JS 파일에 `node --check`를 건다.
5. 아래 입력 흐름을 함께 확인한다.
   - `새 게임`
   - `이어하기`
   - `Esc`
   - `M / S / L`
   - 패널이 열린 상태에서 대사 진행
6. 필요하면 브라우저 QA로 실제 진행까지 확인한다.

## 먼저 쓰는 스킬

- `gsd-runtime-ui`

## 같이 붙기 쉬운 스킬

- `gsd-release-checks`
