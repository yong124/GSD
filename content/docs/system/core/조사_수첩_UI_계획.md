# 조사 수첩 UI 계획

## 1. 목적

경성뎐은 단순 비주얼노벨이 아니라, `조사`, `단서`, `관계`, `공명`, `기록`을 관리하며 진실에 접근하는 조사형 내러티브 게임으로 확장하려 한다.

현재 내부 시스템은 일부 구현돼 있지만, 플레이어가 이를 `보는 UI`가 부족하다.

따라서 이번 UI 확장의 목적은 다음과 같다.

- 현재 상태를 플레이어가 읽을 수 있게 만든다
- 인물과 단서를 한곳에서 정리할 수 있게 만든다
- 추리 퍼즐 확장용 인터페이스 기반을 만든다

이 목적을 위해 1차 중심 UI를 `조사 수첩 패널`로 정의한다.

---

## 2. 경성뎐의 UI 역할 정의

조사 수첩은 단순한 메모장이나 백과사전이 아니다.

역할은 4가지다.

1. `상태 표시`
- 공명, 신뢰, 조사 진행 같은 현재 추적 상태를 보여준다

2. `정보 정리`
- 인물과 단서를 정리하고, 플레이어가 사건 정보를 다시 확인하게 만든다

3. `질문 유지`
- 아직 답이 나지 않은 핵심 질문을 명시적으로 붙들게 만든다

4. `추리 확장 기반`
- 나중에 단서 연결 / 질문 해결 / 논리 퍼즐 UI로 확장할 중심 허브가 된다

---

## 3. 1차 UI 구조

조사 수첩 패널은 4탭 구조를 기본으로 한다.

### 1. 상태

현재 플레이 상태 요약.

표시 항목:

- 공명
- 신뢰
- 조사 진행
- 현재 목표

### 2. 인물

사건과 관련된 주요 인물 정리.

표시 항목:

- 인물명
- 역할
- 현재 인상
- 확인된 사실
- 관련 단서

### 3. 단서

현재 메모장을 확장한 정보 탭.

표시 항목:

- 단서 카테고리
- 신규 획득 여부
- 단서 설명
- 관련 인물 / 장소 연결 여지

### 4. 질문

현재 풀리지 않은 핵심 질문 정리.

표시 항목:

- 질문 목록
- 해결 여부
- 관련 단서 수

---

## 4. 탭별 상세 기획

## 4-1. 상태 탭

이 탭은 숫자를 노출하기보다 `서술형 상태`로 보여주는 것이 핵심이다.

### 공명

내부 값:

- `ResonanceLevel`

표시 단계 예시:

- `안정`
- `전조`
- `심화`
- `침식`

### 신뢰

내부 값:

- `SongsoonTrust`
- `TrustedSongsoon`

표시 단계 예시:

- `경계`
- `동행`
- `신뢰`

### 조사 진행

내부 값:

- `InvestigationScore`
- `ReadRitualScore`

표시 단계 예시:

- `초기`
- `추적`
- `접근`
- `심층`

### 현재 목표

기존 `goal_text`, `goal_kicker`, HUD 정보를 요약해서 붙인다.

---

## 4-2. 인물 탭

이 탭은 추리 퍼즐 확장 시 가장 중요한 축이다.

### 1차 대상 인물

- 유웅룡
- 송순
- 이판규
- 옥련
- 이해심
- 송금
- 편집장

### 표시 항목 예시

- 이름
- 역할
- 현재 상태
- 파악된 사실 2~4줄
- 관련 단서

### 1차 운영 원칙

- 처음엔 정적/반정적 데이터로 시작
- 이후 플래그/단서 획득에 따라 문장이 추가되게 확장

---

## 4-3. 단서 탭

현재 메모장을 `조사 정리형 단서 탭`으로 승격한다.

### 현재 유지 요소

- 카테고리 구분
- 신규 단서 배지

### 1차 확장 요소

- 단서별 관련 인물 표시
- 단서별 관련 장소 표시
- 중요 단서 강조

### 카테고리 예시

- 의례와 공명
- 기록과 기사
- 현장 물증

---

## 4-4. 질문 탭

이 탭은 1차에선 최소 기능으로 둔다.

### 역할

- 사건의 핵심 질문을 플레이어가 잊지 않게 만든다
- 추리 퍼즐 시스템의 미래 진입점이 된다

### 예시 질문

- 송금은 왜 사라졌는가
- 낙원의 의식은 누가 주도했는가
- 이판규는 누구에게 불려갔는가
- 공명은 실재인가, 연출된 장치인가

### 1차 구현

- 질문 목록 노출
- 해결 여부 표기 여지 확보

### 2차 확장

- 관련 단서 연결
- 질문 해결 체크
- 추론 퍼즐 해금

---

## 5. UI 배치 방향

조사 수첩은 기존 메모장 패널을 발전시키는 방향으로 간다.

기본 방향:

- 우측 패널 또는 전체 오버레이형 수첩
- 상단 탭: `상태 / 인물 / 단서 / 질문`
- 현재 대화 화면을 완전히 가리지 않되, 읽기 가능한 정보 밀도를 확보

### 1차 추천안

- 기존 메모장 위치 활용
- 패널 상단 탭 추가
- 현재 단서 UI를 `단서 탭`으로 이동
- 상태 탭과 인물 탭을 새로 추가

---

## 6. 데이터 준비 항목

## 6-1. 즉시 필요한 것

- 상태값을 단계 텍스트로 변환하는 규칙
- 인물 요약 데이터
- 질문 목록 데이터

## 6-2. 이후 필요한 것

- 단서-인물 연결
- 단서-질문 연결
- 질문 해결 조건
- 추론 조합 규칙

---

## 7. 구현 단계

### 1단계

- 패널 명칭을 메모장에서 조사 수첩으로 재정의
- 탭 UI 추가
- 상태 탭 구현

### 2단계

- 인물 탭 기본 구현
- 단서 탭 구조 정리

### 3단계

- 질문 탭 추가
- 질문 데이터 연결

### 4단계

- 추론/연결 UI로 확장
- 단서 연결, 질문 해결, 퍼즐 판정 로직 추가

---

## 8. 우선순위

가장 먼저 구현해야 할 것은 다음 순서다.

1. `상태 탭`
2. `인물 탭`
3. `단서 탭 재정리`
4. `질문 탭 뼈대`

이 순서가 좋은 이유는, 기존 시스템을 가장 적은 비용으로 `보이는 게임성`으로 전환할 수 있기 때문이다.

---

## 9. 결론

경성뎐의 다음 UI 확장은 새로운 숫자 게이지를 더 만드는 것이 아니다.

핵심은:

- 현재 상태를 읽게 하고
- 인물과 단서를 정리하게 하고
- 아직 답이 없는 질문을 붙들게 만드는 것

즉, `조사 수첩 패널`은 추리 퍼즐 게임으로 확장되기 위한 첫 번째 핵심 UI다.
