# 경성뎐 UI/UX 개선 계획

## 1. 현재 상태 요약

현재 게임 화면은 다음 강점을 갖고 있다.

- 타이틀 화면의 무드와 톤이 분명하다.
- 배경, 스탠딩, 포커스, FX가 붙으면서 장면 존재감이 살아났다.
- 대화창, 선택지, 메모장, 저장 패널 등 기본 플레이 UI는 갖춰져 있다.

반면 플레이 체감에서 아쉬운 점도 분명하다.

- 플레이어가 "지금 무엇을 하고 있는가"가 화면에 충분히 드러나지 않는다.
- 선택 결과가 데이터상으로는 반영돼도 화면상 즉시 보이는 피드백은 아직 약하다.
- 조사 우선순위 시스템이 들어왔지만, 일반 선택지와 시각적으로 크게 구분되지 않는다.
- 메모장, 저장 슬롯, 장면 정보가 기능은 있어도 UX 우선순위가 낮아 보인다.
- 조사형 게임의 리듬보다 "대사창을 넘기는 경험"이 아직 더 강하다.

즉, 현재의 과제는 미감 보강보다도 "플레이 정보와 행동 의미를 UI로 번역하는 것"이다.

## 2. 목표

UI/UX 개선 목표는 아래 4가지로 잡는다.

1. 플레이어가 현재 상태를 한눈에 이해하게 만든다.
2. 선택과 조사의 차이를 화면 차원에서 분명히 보이게 만든다.
3. 선택 직후 결과가 즉시 체감되도록 피드백을 강화한다.
4. 포트폴리오에서 "텍스트 게임"이 아니라 "조사 시스템이 있는 게임"으로 보이게 만든다.

## 3. 우선순위별 개선안

### A. 즉시 적용 가치가 큰 개선

#### A-1. 조사 모드 UI 분리 강화

현재 `priority_budget` 씬도 일반 선택지와 비슷하게 보인다.

개선 방향:

- 조사 씬에서 선택지 박스 상단에 `조사 중`, `남은 조사 기회`, `현재 장면 목적`을 표시한다.
- 조사 선택지는 일반 선택지보다 카드형에 가깝게 보이도록 높이와 패딩을 늘린다.
- 이미 고른 항목은 `조사 완료`, 아직 안 고른 항목은 `미조사` 상태로 구분한다.
- 조사 종료 후 `이번 장면에서 확인한 것` 짧은 요약 한 줄을 띄운다.

기대 효과:

- 플레이어가 지금 "대화 선택"이 아니라 "탐문/조사"를 하고 있다는 점을 즉시 이해한다.

#### A-2. 선택 결과 피드백 UI 추가

지금은 관계, 조사, 공명 변화가 대사로는 반영되지만 UI 피드백은 약하다.

개선 방향:

- 선택 직후 작은 시스템 토스트를 띄운다.
- 예:
  - `송순과의 거리 변화`
  - `단서 감각이 또렷해짐`
  - `공명 심화`
  - `기록 의지 강화`
- 수치 노출은 최소화하고, 의미 중심 표현을 쓴다.

기대 효과:

- 플레이어가 자신의 선택이 실제 상태를 바꿨다는 감각을 바로 받는다.

#### A-3. 메모장 UX 승격

현재 메모장은 존재하지만 보조 기능처럼 보인다.

개선 방향:

- 메모장 버튼에 신규 단서 획득 시 점 표식 배지 추가
- 메모장 내부를 `단서`, `기사/기록`, `인물` 정도로 구분할 수 있게 확장 검토
- 현재 장면과 관련 있는 단서는 상단 고정 또는 강조

기대 효과:

- 증거 시스템이 장식이 아니라 플레이 핵심 축으로 보인다.

### B. 게임성 체감에 큰 도움이 되는 개선

#### B-1. 장면 목적 표시

지금은 플레이어가 무엇을 위해 이 장면을 플레이하는지 흐려질 때가 있다.

개선 방향:

- 씬 진입 시 1~2초간 짧은 목표 문구를 보여준다.
- 예:
  - `송순의 진술에서 모순 찾기`
  - `의식 기록의 순서 확인`
  - `우물터 흔적 조사`

기대 효과:

- 조사형 게임의 리듬이 선명해진다.

#### B-2. 상태 변화 시각화

현재 `SongsoonTrust`, `InvestigationScore`, `ResonanceLevel`이 데이터상 존재한다.

개선 방향:

- 수치를 노출하지 않더라도 상태 방향은 UI에서 보여준다.
- 예:
  - 송순 신뢰 상승 시 스탠딩 거리감/포커스 완화
  - 공명 상승 시 배경 비네트/블루 FX 미세 강화
  - 조사 진척 시 메모장/장면 목표 강조 방식 변화

기대 효과:

- 숫자를 숨긴 채로도 플레이 스타일이 축적된다는 느낌을 준다.

#### B-3. 선택지 정보 계층 분리

모든 선택지가 같은 무게로 보이면 조사 게임 체감이 약해진다.

개선 방향:

- 일반 선택지, 조사 선택지, 위험 선택지, 관계 선택지를 시각적으로 다르게 처리
- 예:
  - 관계 선택: 부드러운 톤
  - 위험 선택: 붉은 기색
  - 조사 선택: 금색/청색 혼합
  - 결정 선택: 두꺼운 테두리

기대 효과:

- 플레이어가 선택 전부터 "이 버튼이 어떤 종류의 판단인지" 읽을 수 있다.

### C. 포트폴리오 효과가 큰 개선

#### C-1. 조사 흐름 HUD

화면 한쪽에 작은 조사 HUD를 둔다.

표시 예:

- 현재 장면
- 남은 조사 기회
- 확보 단서 수
- 현재 조사 축

주의:

- 전면 HUD가 아니라, 몰입을 깨지 않는 최소형이어야 한다.

기대 효과:

- 이 프로젝트가 비주얼노벨이 아니라 시스템 게임이라는 인상을 강하게 준다.

#### C-2. 장면 전환 카드 고급화

현재 챕터 카드와 씬 배너는 있지만 활용도가 더 높아질 수 있다.

개선 방향:

- 조사 장면 진입 시 `장소 + 시간대 + 목적` 카드
- 후반 의식 진입 시 카드 애니메이션을 조금 더 강하게
- 엔딩 직전은 더 건조하고 무겁게

기대 효과:

- 챕터/씬 구조가 더 영화적으로 읽힌다.

#### C-3. 입력 힌트/진행 리듬 정리

현재 `▼` 힌트는 있으나 인터랙션 체계가 약간 낡아 보일 수 있다.

개선 방향:

- 클릭/스페이스/엔터 힌트를 통합
- 선택지 등장 전후의 전환 리듬을 조금 더 명확하게
- 저장/메모장/불러오기 버튼에도 hover/focus 상태를 더 분명하게 부여

기대 효과:

- 기본 완성도가 올라간다.

## 4. 추천 실행 순서

### 1단계. 저비용 고효율

- 조사 모드 UI 분리 강화
- 선택 결과 시스템 토스트
- 메모장 신규 단서 배지
- 버튼/선택지 hover/focus 정리

이 단계만 해도 플레이어가 "시스템이 있다"는 인상을 크게 받는다.

### 2단계. 게임성 번역

- 장면 목적 표시
- 선택지 유형별 스타일 분리
- 상태 변화의 시각적 피드백 연결

이 단계부터는 조사형 게임으로서의 체감이 확실히 올라간다.

### 3단계. 포트폴리오 마감

- 조사 HUD
- 장면 전환 카드 고급화
- 저장/메모장/시스템 패널 폴리시

이 단계는 면접/포트폴리오에서 보여주는 인상이 크게 좋아진다.

## 5. 내일 바로 시작할 작업 추천

다음 작업 시작점은 아래 순서를 권장한다.

1. `priority_budget` 씬 전용 조사 헤더 UI 추가
2. 선택 직후 상태 토스트 추가
3. 메모장 신규 단서 배지 추가
4. 장면 목표 문구 시스템 추가

이 순서가 좋은 이유는 구현 난이도 대비 체감이 크고, 현재 데이터 구조와도 잘 맞기 때문이다.

## 6. 한 줄 정리

현재 경성뎐 UI/UX의 핵심 과제는 "더 화려하게 만드는 것"이 아니라 "플레이어가 지금 어떤 판단을 하고 있고, 그 결과가 어떻게 남는지를 화면에서 바로 읽게 만드는 것"이다.
