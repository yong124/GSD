## 1차. 기존 테이블 확장 + EditorNode 선반영

### 목표

새 시트를 크게 늘리기 전에, 기존 구조에 자연스럽게 흡수 가능한 데이터를 먼저 이동시킨다.

### 범위

#### xlsx

- `SceneTable`
  - `GoalKicker`
  - `GoalText`
- `CharacterTable`
  - `RoleText`
  - `NotebookSummary1`
  - `NotebookSummary2`
- `EvidenceTable`
  - `CategoryID`
  - `CategoryTitle`
  - `CategoryHint`

#### EditorNode

- Scene 패널에 목표 필드 추가
- Character 패널에 수첩 요약 필드 추가
- Evidence 패널에 카테고리 필드 추가
- normalize/export 구조에 새 필드 보존

#### json 데이터

- `game_data.js`에서 위 필드를 수용 가능하게 구조 정리

#### 런타임

- 조사 수첩에서
  - `goal_kicker`, `goal_text`
  - character notebook fields
  - evidence category fields
  를 읽도록 전환 준비

### 완료 기준

- EditorNode에서 1차 필드가 전부 편집 가능
- export 시 새 필드가 떨어지지 않음
- 기존 기능이 깨지지 않음

---

## 2차. QuestionTable 도입

### 목표

질문 탭 하드코딩을 제거하고, 조사 수첩 질문 구조를 독립 메타 데이터로 분리한다.

### 범위

#### xlsx

- `QuestionTable` 신설

권장 컬럼:

- `QuestionID`
- `Title`
- `Detail`
- `SortOrder`
- `Category`
- `VisibleRuleID`
- `StateRuleID`

#### EditorNode

- `questions` 전역 메타 구조 추가
- 질문 편집 UI 추가
- 1차는 캐릭터 탭 하단 또는 별도 섹션

#### 파이프라인

- `export_to_json.py`
- `json_to_generated_xlsx.py`
- `validate_game_data.py`

질문 시트를 읽고/generated로 다시 뽑을 수 있게 확장

#### json 데이터

- `game_data.questions` 추가

#### 런타임

- `getQuestionEntries()` 하드코딩 제거
- `QuestionTable` 기준 렌더로 교체

### 완료 기준

- 질문 탭이 코드 상수 없이 데이터 기반으로 동작
- 질문 노출/상태 구조가 최소한 RuleID 기반 참조를 가짐

---

## 3차. 상태 문구/규칙 일반화

### 목표

공명, 신뢰, 조사 상태 해석과 노출 규칙을 공통 데이터 규칙으로 전환한다.

### 범위

#### xlsx

- `StateDescriptorTable` 신설
- 필요하면 `RuleTable` 검토

#### EditorNode

- 상태 해석 데이터 편집 UI
- 필요 시 `조사 메타` 탭 분리

#### 파이프라인

- descriptor/rule 시트 왕복 지원

#### json 데이터

- `state_descriptors`
- 필요 시 `rules`

#### 런타임

- `getResonanceState()`
- `getTrustState()`
- `getInvestigationState()`
  하드코딩 제거

### 완료 기준

- HUD와 조사 수첩이 같은 상태 해석 데이터를 공유
- 조건식이 반복될 경우 공통 규칙 구조로 흡수 가능
