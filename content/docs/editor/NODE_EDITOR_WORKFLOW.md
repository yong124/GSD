# 경성뎐 Node Editor 작업 루틴

본 문서는 `EditorNode`를 메인 작업 툴로 사용할 때의 기본 작업 루틴을 정리한 문서다.

---

## 1. 기본 원칙

- 메인 편집은 `EditorNode`에서 진행한다.
- 구조 수정은 가능한 한 `EditorNode`에서 끝낸다.
- `xlsx`는 대량 편집 / 검수 / 공유가 필요할 때만 사용한다.
- `script.xlsx`는 자동 덮어쓰기하지 않는다.

---

## 2. 기본 작업 흐름

1. `EditorNode`에서 씬 / 대사 / 선택지 / 분기 / 단서를 수정한다.
2. 노드 연결, 대사 프리뷰, 구조 검수 패널로 기본 이상 여부를 확인한다.
3. `내보내기`로 최신 `game_data.js`를 저장한다.
4. 테이블 검수나 대량 수정이 필요하면 별도로 `script.generated.xlsx`를 생성한다.
5. 필요한 시트만 확인해서 `script.xlsx`에 수동 반영한다.

---

## 3. xlsx를 사용하는 경우

- 대사 텍스트 대량 수정
- 특정 컬럼 일괄 정리
- 외부 검수 공유
- 표 기반 확인이 더 편한 경우

위 경우가 아니면 기본적으로 `EditorNode`를 우선 사용한다.

---

## 4. 주의 사항

- `script.xlsx`는 양식 보존 이슈 때문에 자동 갱신하지 않는다.
- generated 파일은 복붙용 산출물로 사용한다.
- 구조 연결 수정은 `xlsx`보다 `EditorNode`에서 처리하는 편이 안전하다.

---

## 5. 한 줄 정리

기본 작업은 `EditorNode`, 대량 수정과 검수는 `xlsx`, 최종 런타임 반영은 `game_data.js` 기준으로 운영한다.
