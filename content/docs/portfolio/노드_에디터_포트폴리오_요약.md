# 경성뎐 Node Editor 포트폴리오 정리

## 1. 문서 목적

본 문서는 `경성뎐` 프로젝트에서 Node Editor를 어떤 문제 해결 과정으로 만들었고, 포트폴리오에서 어떤 강점으로 설명할 수 있는지 정리한 문서다.

---

## 2. 왜 만들었는가

초기에는 하드코딩과 테이블 중심 작업으로도 콘텐츠를 운영할 수 있었다.

하지만 프로젝트가 커지면서 아래 문제가 명확해졌다.

- 씬 연결 구조를 한눈에 보기 어렵다
- 선택지와 분기의 관계를 추적하기 번거롭다
- 대사 수정과 구조 검토를 다른 화면에서 번갈아 해야 한다
- 기획 검수 비용이 커진다

Node Editor는 이 문제를 해결하기 위한 기획 작업 도구로 확장되었다.

---

## 3. 발전 흐름

```mermaid
flowchart LR
  A["하드코딩 프로토타입"] --> B["xlsx 기반 데이터 분리"]
  B --> C["xlsx -> game_data 파이프라인"]
  C --> D["전용 편집기"]
  D --> E["Node Editor"]
  E --> F["검수 / 시각화 / 준대량 편집"]
```

---

## 4. 왜 xlsx를 보조 포맷으로 남겼는가

Node Editor가 메인 툴이 되더라도 xlsx는 여전히 의미가 있다.

이유는 다음과 같다.

- 대량 텍스트 수정이 표 기반에서 더 빠를 때가 있다
- 공유와 검수는 표 형태가 더 편할 때가 있다
- 백업 / 비교 / 전달용 포맷이 필요하다

즉, xlsx를 버리지 않은 이유는 구식이라서가 아니라, `작업 목적이 다른 보조 포맷`으로 남겨두는 것이 더 실무적이기 때문이다.

---

## 5. Node Editor의 기획 툴로서의 강점

### 5-1. 구조 이해

- 씬 연결을 한눈에 파악 가능
- 선택지 / 분기 흐름 시각 확인 가능
- 엔딩 / 미참조 씬 확인 가능

### 5-2. 검수 효율

- 끊긴 링크 검출
- 중복 ID 검출
- 플래그 읽기 / 쓰기 구조 확인
- 대사 프리뷰 확인

### 5-3. 운영 효율

- 구조 수정과 데이터 수정을 같은 도구 안에서 처리
- 반복 수정 시 맥락 전환 감소
- 준대량 편집 기능으로 xlsx 의존도 완화

---

## 6. 포트폴리오에서 강조할 문장

`경성뎐` 프로젝트에서는 ADV 콘텐츠를 단순히 작성하는 데서 그치지 않고, 내러티브 데이터를 운영 가능한 구조로 분리한 뒤, 실제 기획 작업에 맞는 Node Editor를 중심으로 수정, 검수, 시각화까지 가능한 작업 환경으로 발전시켰다.

---

## 7. 한 줄 요약

Node Editor는 `경성뎐` 프로젝트에서 분기형 내러티브 콘텐츠를 더 쉽게 설계하고 검수하기 위해, 실제 제작 문제를 해결하며 확장된 기획 작업 도구다.

---

## 8. 포트폴리오에서 이 문서를 어떻게 써야 하는가

이 문서는 단순히 "에디터를 만들었다"는 설명용이 아니라, 기획자가 실제 콘텐츠 제작 환경까지 설계했다는 증거 문서로 사용하는 것이 좋다.

핵심 메시지는 아래처럼 잡으면 된다.

- 콘텐츠를 직접 작성하는 사람의 작업 흐름을 이해하고 있었다.
- 데이터 구조와 편집 도구를 함께 설계했다.
- 반복 수정 비용을 줄이기 위해 에디터와 검증 구조를 만들었다.
- 결과적으로 콘텐츠 생산성과 유지보수성이 같이 올라갔다.

## 9. 장표에서 강조할 화면 우선순위

포트폴리오 장표에는 아래 순서로 화면을 보여주는 것이 가장 설득력 있다.

- 노드 연결과 분기 구조가 보이는 메인 에디터 화면
- 선택한 데이터가 우측 패널에서 수정되는 화면
- 조건 / 상태 / 게이지 등 실제 시스템 데이터가 연결된 화면
- 검증, 미리보기, 이미지 확인 등 실무 편의 기능 화면
- 최종 게임 적용 장면

즉, 게임 결과 화면은 "마무리 증거"로 쓰고, 에디터 화면은 "핵심 역량 증거"로 써야 한다.

## 10. 추가로 확보하면 좋은 증거

아래 자료가 더 있으면 이 문서의 힘이 더 세진다.

- 초기 데이터 구조와 현재 구조 비교 이미지
- 수기 작업 방식과 에디터 작업 방식의 비교
- 작업 시간 단축이나 수정 흐름 단축 예시
- 잘못된 데이터가 검증에서 잡히는 예시
- 실제 콘텐츠가 에디터를 통해 반영되는 전후 장면
