# 경성뎐 다음 스레드 인수인계

## 1. 현재 상태 한 줄 요약

지금 경성뎐은 `분위기형 비주얼노벨 프로토타입` 단계를 넘어서,  
`조사 수첩 + 질문 해결 + 단서 연결 + 증거 제시 + 모순 판별 + 후반 분기 회수`가 실제 플레이 루프로 붙은 상태다.

즉 다음 스레드의 초점은 새 구조를 까는 게 아니라:

- 추리 루프를 더 많은 씬에 확장
- 질문/모순 해결 결과를 중반 행동과 후반 엔딩에 더 깊게 연결
- 추리 퍼즐 게임성 체감을 더 올리는 것

이다.

## 3. 추리 게임성 쪽에서 지금까지 붙은 것

### 3-1. 질문 해결

질문 탭에서:

- 질문 선택
- 관련 단서 확인
- 단서 제출
- 해결 상태 갱신

이 실제로 동작한다.

주요 필드:

- `ResolutionType`
- `RelatedEvidenceIDs`
- `SolutionEvidenceIDs`
- `SolutionMode`
- `SolvedFlagID`
- `RewardFlagID`

### 3-2. 단서 연결

질문에 따라 단일 단서 제출이 아니라:

- 복수 단서를 고른 뒤
- 함께 제출해야 해결되는

`단서 연결형 질문`도 들어가 있다.

현재 대표 예:

- `QRitualLead`

### 3-3. 증거 제시

씬 안에서 `보유 단서를 들이민다`는 루프가 실제로 붙어 있다.

현재 적용 샘플:

- `ch4a_library`
- `ch5_ritual_room`

즉 지금은

- 질문 해결
- 단서 연결
- 증거 제시

가 각기 따로 있는 게 아니라, 수첩과 현장 행동이 이어지는 구조다.

### 3-4. 모순 판별

`Contradiction` 타입 질문이 들어가 있다.

현재 질문 탭에서:

- 반박 문장 표시
- 적절한 단서 제출
- `해결됨` 전환

이 실제로 동작한다.

현재 예:

- `QIpangyuMadness`
- `QSonggeumRunaway`
- `QRitualAccident`

### 3-5. 후반 회수

질문/모순 해결 결과는 이제 후반에서 실제로 회수된다.

대표 회수 씬:

- `ch6_threshold`
- `ch6_outcome`
- `ch6_path_a2`
- `ch6_path_b2`
- `ch6_ending_a`
- `ch6_ending_b`
- `ch6_ending_c`
- `ch6_epilogue`

그리고 최근엔 여기서 더 나아가:

- 질문/모순 해결로 `전용 행동`이 해금되고
- 그 행동이 다시 후반 엔딩 경로를 더 직접 기울이는 구조

까지 붙였다.

최근 추가 플래그:

- `ExposedArchivePattern`
- `MatchedRitualPattern`
- `ExposedTruthAtRitual`

즉 지금은 `질문을 푸는 것`이 수첩 안에서 끝나지 않고,

- 중반 새 선택지
- 중반 새 증거 제시
- 최종 priority 행동
- 엔딩 경로 / 엔딩 문장

으로 이어진다.

## 7. 다음 스레드에서 바로 할 일

### 1순위. 질문/모순 해결 결과가 새 증거 제시를 더 여는 구조

지금은:

- 질문 해결 -> 새 선택지
- 질문 해결 -> 새 priority 행동
- 질문 해결 -> 새 후반 문장

쪽이 많이 붙어 있다.

다음은 여기서 더 나아가:

- 질문 해결 -> 중반 특정 씬에서 새 `evidence_choice` 해금

을 더 늘리는 게 제일 자연스럽다.

추천 적용 후보:

- `ch3_warehouse`
- `ch4a_library`
- `ch5_ritual_room`

### 2순위. 추리 루프를 중반 씬에 더 확장

현재 추리 루프는 분명히 들어갔지만, 아직 대표 샘플 중심이다.

다음엔:

- 질문 1~2개 추가
- 모순 판별 1개 추가
- 증거 제시 적용 씬 1~2개 추가

정도로 게임 전체에 더 퍼뜨리면 좋다.

### 3순위. 분기/엔딩 차등을 더 강하게

지금도 회수는 되지만, 앞으로는:

- 특정 질문 해결 조합일 때만 열리는 행동
- 특정 모순 판별 성공 시만 열리는 전용 장면
- 엔딩 A/B/C 내부 문장 차이를 더 크게

로 키울 수 있다.
