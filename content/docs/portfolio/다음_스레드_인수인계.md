# 경성뎐 다음 스레드 인수인계

## 1. 현재 상태 한 줄 요약

지금 경성뎐은 단순 비주얼노벨 프로토타입 단계를 넘어서, 아래 4축이 실제로 연결된 상태다.

- `캐릭터/감정/스테이징 데이터 구조`
- `조사 우선순위 시스템`
- `자체 런타임 + 3슬롯 저장`
- `EditorNode + generated xlsx + validate` 파이프라인

즉 지금부터의 작업은 "구조를 새로 깔기"보다 "플레이 체감과 폴리시를 마감하는 단계"에 가깝다.

## 2. 현재 워크트리에서 실제로 들어간 큰 기능

### 2-1. 캐릭터 / 감정 / 스테이징 구조

`game/data/game_data.js`에는 아래가 실제 반영돼 있다.

- `characters`
- `character_emotions`
- 대사별 `SpeakerID`
- 대사별 `EmotionType`
- 대사별 `StandingSlot`
- 대사별 `FocusType`
- 대사별 `EnterMotion`
- 대사별 `ExitMotion`
- 대사별 `IdleMotion`
- 대사별 `FxType`

현재 기준:

- `CharacterTable 15행`
- `CharacterEmotionTable 29행`
- 화자 대사 239줄에 `SpeakerID / EmotionType / StandingSlot / FocusType` 반영

### 2-2. 런타임 연출 시스템

다음이 실제 게임 화면에서 동작한다.

- 좌/중/우 스탠딩 배치
- 화자 강조 / 비화자 dim
- 기본 입장 모션 / 흔들림
- 대사 단위 FX
  - `BlueTrace`
  - `RitualGlow`
  - `BloodSmear`
  - `Flicker`

관련 파일:

- `game/index.html`
- `game/css/main.css`
- `game/css/dialogue.css`
- `game/css/effects.css`
- `game/js/engine/dialogue.js`
- `game/js/engine/effects.js`

### 2-3. 조사 우선순위 시스템

엔진과 데이터 둘 다 들어가 있다.

현재 적용 씬:

- `ch2_cafe`
- `ch3_room4`
- `ch6_ritual_scene`

현재 흐름:

1. 조사 기회 표시
2. 1개 선택
3. 분기 대사 재생
4. 남은 선택지 재표시
5. 예산 소진 후 `priority_after_dialogues`
6. 다음 씬 이동

관련 파일:

- `game/js/engine/choice.js`
- `game/js/engine/scene.js`
- `game/data/game_data.js`

### 2-4. 선택 후 반응 대사 구조

일반 선택지 씬들도 이제 거의 전부 아래 구조다.

- `선택`
- `짧은 반응 대사`
- `다음 분기 씬 본문`

즉 예전처럼 `선택 -> 바로 점프`되는 감각은 많이 줄어든 상태다.

### 2-5. 단서 시스템 연결 강화

단서 수집 시 이제 아래가 같이 반영된다.

- `State.evidence` 저장
- `HasEvidence_<EvidenceID>` 플래그 세팅

그리고 아래 씬들엔 단서 기반 반응 대사가 붙어 있다.

- `ch4a_library`
- `ch5_ritual_room`
- `ch6_threshold`

관련 파일:

- `game/js/engine/evidence.js`
- `game/data/game_data.js`

### 2-6. 멀티 슬롯 저장

단일 저장이 아니라 `3슬롯 저장/불러오기`가 들어가 있다.

현재 되는 것:

- 슬롯 1~3 저장
- 슬롯별 씬/챕터/시간 표시
- 마지막 사용 슬롯 기준 자동저장
- 구버전 단일 저장 마이그레이션

관련 파일:

- `game/js/engine/save.js`
- `game/index.html`
- `game/css/main.css`

### 2-7. EditorNode 확장

에디터는 지금 아래 수준까지 확장됐다.

- `노드 편집` / `캐릭터 편집` 탭 분리

- `CharacterEmotionTable` 편집
- `SpeakerID`, `EmotionType`, `StandingSlot`, `FocusType`, `Motion`, `Fx` 편집
- enum 필드 드롭다운

관련 파일:

- `EditorNode/index.html`
- `EditorNode/editor.css`
- `EditorNode/editor.js`

## 4. 한글 관련 주의

터미널에서 한글이 깨져 보이면 바로 파일 손상으로 판단하지 말 것.

PowerShell에서는 아래 조합으로 먼저 확인한다.

```powershell
chcp 65001 > $null
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
Get-Content '경로' -Encoding utf8
```

즉:

- 출력 깨짐 = 우선 환경 문제 의심
- 실제 손상 여부는 UTF-8 기준 재확인 후 판단

주의:

- 이번 스레드에서 `shell` 경유로 대사를 넣다가 `game_data.js` 일부 한글이 실제 `???`로 손상된 적이 있었다.
- 이후 복구했고, 지금은 `validate`와 UTF-8 재확인 기준으로 정상 상태다.
- 앞으로 한국어 데이터 수정은 가능하면 `apply_patch` 위주로 가는 편이 안전하다.

## 6. 현재 기준으로 완료 / QA 필요 / 미완

### 완료됨

- 캐릭터/감정 데이터 구조
- 스탠딩/포커스/기본 FX 런타임
- 3슬롯 저장/불러오기
- 파이프라인 확장
- EditorNode 캐릭터 탭 분리
- enum 드롭다운
- UI/UX 1차, 2차 기본 골격

### 들어갔지만 QA 더 필요

- `priority_dialogues` 에디터 UI 실사용 QA
- 조사 우선순위 씬 체감 QA
- HUD / 목표 카드 / 메모장 배지의 실제 화면 균형
- 스탠딩/포커스/모션 과한 씬과 약한 씬 튜닝

### 아직 미완

- `priority` 시스템을 더 많은 씬으로 확장
- `evidence`가 분기와 엔딩에 더 직접 영향을 주는 연결
- 저장/메모장/패널 폴리시 마감
- 실제 감정별 스탠딩 에셋 적용
- 작은 해상도/브라우저 실플레이 반응형 QA

## 7. 다음 스레드 우선순위

### 1순위. UI/UX 3차 폴리시

추천 순서:

1. 조사 HUD에 `남은 조사 기회`와 현재 목표를 더 직접 연결
2. 저장 패널 / 메모장 패널 시각 정리
3. 작은 해상도에서 HUD, 목표 카드, 선택지 겹침 없는지 반응형 튜닝

### 2순위. priority 실사용 QA

특히 아래 확인 필요:

- `ch2_cafe`
- `ch3_room4`
- `ch6_ritual_scene`

볼 것:

- 조사 헤더 체감
- 남은 조사 기회 표시
- 분기 대사 후 재표시 리듬
- `priority_after_dialogues` 전환감

### 3순위. 후반 데이터 폴리시 마감

특히:

- `ch5_ritual_path`
- `ch6_ending_a`
- `ch6_ending_b`

- `ch6_epilogue`

포인트:

- 엔딩별 여운 차이 더 분명하게

### 4순위. evidence 시스템의 게임성 강화

다음 단계 후보:

- 특정 단서가 있어야 열리는 반응 대사 더 확대
- 단서 조합 기반 분기
- 엔딩 조건에서 단서의 가중치 반영

## 9. 한 줄 결론

지금 경성뎐은 `구조를 만드는 단계`는 거의 넘겼고, 다음 스레드 핵심은 `실플레이 UX와 조사 게임 체감`을 마감 감각으로 밀어붙이는 것이다.
