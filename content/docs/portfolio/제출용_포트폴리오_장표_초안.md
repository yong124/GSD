# 제출용 포트폴리오 페이지 구성 초안

## 1. 문서 목적

이 문서는 발표용 슬라이드 초안이 아니라, `읽는 문서형 PPT 포트폴리오`를 만들기 위한 페이지 구성 초안이다.

즉 목표는 아래와 같다.

- PPT처럼 보이지만 문서처럼 읽힌다
- 기획 의도와 구조가 분명하게 드러난다
- 시각 자료가 많아 읽기 부담이 낮다
- 페이지 단위로 완결성이 있다

추천 분량은 12페이지다.

---

## 2. 전체 구성

### Page 1. 표지

#### 목적

이 포트폴리오가 단순한 게임 소개가 아니라, 시스템 기획과 제작 구조 설계 중심의 문서라는 점을 첫 페이지에서 분명히 보여준다.

#### 넣을 내용

- 프로젝트명
- 한 줄 포지셔닝
- 지원 포지션
- 이름

#### 추천 시각 자료

- 에디터 메인 화면 일부
- 게임 화면 일부를 작게 보조로 배치

#### 배치 방식

- 상단: 제목
- 중앙: 에디터 캡처 크게
- 하단: 한 줄 설명과 기본 정보

---

### Page 2. 프로젝트 개요

#### 목적

이 프로젝트가 어떤 프로젝트였고, 어디까지 확장된 작업인지를 짧고 분명하게 설명한다.

#### 넣을 내용

- 브라우저 기반 ADV 프로젝트
- 초기에는 하드코딩 중심 구조로 시작
- 이후 데이터 구조, 에디터, 검증 흐름까지 확장
- 결과적으로 제작 환경 자체를 정리한 프로젝트

#### 추천 시각 자료

- 게임 화면
- 에디터 화면

#### 배치 방식

- 좌측: 설명 문단
- 우측: 게임 / 에디터 비교 이미지
- 하단: 핵심 문장 박스

---

### Page 3. 문제 정의

#### 목적

왜 이 프로젝트에서 구조 정리와 에디터 설계가 필요했는지를 설명한다.

#### 넣을 내용

- 콘텐츠가 늘어날수록 수정 비용이 증가했다
- 분기와 상태 변화가 코드 안에 퍼져 있었다
- 기획 검증 속도가 느렸다
- 작업자가 직접 수정하기 어려운 구조였다

#### 추천 시각 자료

- Before 구조도
- 초기 데이터 흐름 예시

#### 배치 방식

- 좌측: 문제 설명 3~4줄
- 우측: Before 구조도
- 하단: 요약 박스

---

### Page 4. 기획 목표

#### 목적

무엇을 만들려고 했는지보다, 어떤 문제를 어떤 방향으로 해결하려 했는지를 보여준다.

#### 넣을 내용

- 콘텐츠를 코드에서 분리해 데이터로 관리
- 기획 개념을 일관된 구조로 정리
- 분기를 시각적으로 다룰 수 있는 에디터 설계
- 검증과 미리보기를 붙여 작업 안정성 확보

#### 추천 시각 자료

- 목표 구조 다이어그램
- 제작 흐름 스케치

#### 배치 방식

- 상단: 제목과 짧은 설명
- 중앙: 구조 다이어그램
- 하단: 목표 4개를 카드형으로 정리

---

### Page 5. 시스템 구조

#### 목적

이 프로젝트를 기획 포트폴리오로 보이게 만드는 핵심 페이지다.

#### 넣을 내용

- 질문, 선택지, 결과 구조
- Condition 중심 분기 구조
- 상태값과 게이지
- 조사, 증거, 인물 상태의 연결 관계

#### 추천 시각 자료

- 시스템 개념도
- 구조 연결도

#### 배치 방식

- 좌측: 구조 설명
- 우측: 시스템 개념도
- 하단: 핵심 포인트 3개

---

### Page 6. 데이터 구조 정리

#### 목적

레거시 개념을 어떻게 정리하고, 현재 구조를 어떤 기준으로 재분류했는지 보여준다.

#### 넣을 내용

- Flag / Rule 중심 개념을 정리한 과정
- Condition 중심 구조로 통합한 이유
- 필요한 데이터는 남기고 의미 없는 레거시는 제거한 기준
- 실제 운영 가능한 테이블 구조로 재정리한 방식

#### 추천 시각 자료

- Before / After 표
- 테이블 구조 비교

#### 배치 방식

- 상단: 설명 문단
- 중앙: 비교 표
- 하단: “왜 이렇게 정리했는가” 요약 박스

---

### Page 7. 에디터 설계

#### 목적

왜 에디터가 필요했고, 어떤 관점으로 설계했는지를 설명한다.

#### 넣을 내용

- 분기를 한눈에 보도록 노드 구조를 채택
- 데이터 패널과 흐름 편집을 연결
- 작업자가 직접 수정 가능한 구조 설계
- 시각적 확인과 수정 흐름을 같이 고려

#### 추천 시각 자료

- 에디터 메인 캡처
- 우측 패널 편집 화면

#### 배치 방식

- 좌측: 에디터 설계 의도
- 우측: 에디터 화면 크게
- 하단: 주요 기능 태그

---

### Page 8. 작업 흐름과 검증 방식

#### 목적

에디터를 만든 것에서 끝나지 않고, 제작 흐름 전체를 어떻게 연결했는지 보여준다.

#### 넣을 내용

- 편집 -> 변환 -> 검증 -> 반영 흐름
- FK / Enum 기반 선택 구조
- 이미지 미리보기와 확인 기능
- 누락값과 잘못된 연결을 잡는 검증 방식

#### 추천 시각 자료

- 작업 파이프라인 흐름도
- 검증 결과 화면

#### 배치 방식

- 상단: 작업 흐름도
- 하단 좌측: 설명 문단
- 하단 우측: 검증 포인트 요약

---

### Page 9. Before / After

#### 목적

프로젝트가 단계적으로 개선돼 왔다는 점을 한눈에 보여준다.

#### 넣을 내용

- 초기: 하드코딩 중심
- 중간: 데이터 구조 분리
- 이후: Condition 중심 통합
- 현재: 에디터 + 검증 + 미리보기 + 반영 흐름

#### 추천 시각 자료

- 전후 비교 표
- 타임라인

#### 배치 방식

- 좌우 비교 또는 단계형 타임라인
- 하단: 변화의 의미를 짧게 정리

---

- 상단: 제목

- 하단: 요약 문장

### Page 11. 회고

#### 목적

이 페이지가 문서형 포트폴리오의 완성도를 크게 올려준다.

#### 넣을 내용

- 잘한 판단
- 아쉬운 판단
- 다음에 보완하고 싶은 점
- 이번 프로젝트에서 가장 크게 배운 점

#### 추천 시각 자료

- 프로젝트 변화 타임라인
- 판단 정리 표

#### 배치 방식

- 좌측: 잘한 점 / 아쉬운 점
- 우측: 다음 보완점 / 배운 점
- 하단: 한 줄 회고

### Page 12. 내가 보여주고 싶은 역량

#### 목적

이 프로젝트를 통해 결국 어떤 기획자로 보이고 싶은지를 마무리한다.

#### 넣을 내용

- 시스템을 구조화해서 정의하는 능력
- 콘텐츠 생산 구조를 설계하는 능력
- 에디터와 데이터 구조를 함께 보는 능력
- AI를 도구로 활용해 제작 속도를 높이는 능력

#### 추천 시각 자료

- 키워드 카드
- 전체 구조 축약 다이어그램

#### 배치 방식

- 상단: 요약 문장
- 중앙: 역량 카드 4개
- 하단: 마무리 문장

## 3. 문서형 포트폴리오에 맞는 페이지 규칙

- 페이지마다 메시지는 하나만 둔다
- 텍스트는 완결성 있게, 하지만 짧게 쓴다
- 시각 자료는 설명을 대신하는 수준으로 크게 쓴다
- 에디터 / 구조 / 검증 화면을 우선한다
- 게임 화면은 보조 증거로 사용한다
